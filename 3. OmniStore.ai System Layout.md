# 3. OmniStore.ai System Layout & Design System

The layout system guarantees a seamless, extremely fast, and completely intuitive User Experience tailored for retail operators (thumb-friendly, straightforward).

## 🧠 STEP 1 — CORE DESIGN PRINCIPLES
1. **Thumb-first design**: All main actions (FABs, sticky bottom save buttons) must be reachable with one thumb.
2. **Big tap targets**: Minimum touch touch area is `48dp`.
3. **High readability**: Big text, high contrast. Optimized for users who may have low digital literacy or vision challenges.
4. **Glanceable UI**: A user should understand the screen in less than 2 seconds.
5. **Minimal colors**: Do not confuse the user. Strict semantic color bounds.

## 🎨 STEP 2 — COLOR SYSTEM
Keep it meaningful and completely avoid gradients to ensure high performance on lower-tier devices.
- **Primary**: Deep Blue or Green (Focuses on trust + finance logic).
- **Semantics**:
  - 🟢 **Green** → Received money, positive actions.
  - 🔴 **Red** → Owed money, destructive actions, negative balances.
  - 🟡 **Yellow** → Warnings (low stock).
  - 🔵 **Blue** → Neutral informative events.

## ✍️ STEP 3 — TYPOGRAPHY
Must use bold for numbers (financial values) and medium weights for standard labels. Fixed visual hierarchy:
- **Headline**: 20–24pt
- **Section Title**: 18pt
- **Body**: 15–16pt
- **Small/Subtext**: 13pt

## 📏 STEP 4 — SPACING & GRID SYSTEM
Relying strictly on a fixed spacing scale prevents a "messy" appearance.
- **Global Spacing Scale**: `4`, `8`, `12`, `16`, `20`, `24`, `32`.
- Examples applied: 
  - `16dp` Screen padding.
  - `12dp` inside card padding.
  - `12dp` between multiple cards horizontally or vertically.

Grid structure for layouts is strictly a **1-column layout** for maximum width spacing. Avoid complex columns on mobile.

## 🔲 STEP 5 — CORE COMPONENTS
Before putting screens together, these are the universal building blocks used across the application:

1. **App Bar**: Left title, right actions, exactly `56dp` high, no clutter.
2. **Bottom Navigation**: Maximum 5 standard destinations (`Home`, `Customers`, `Inventory`, `Records`, `Assistant`).
3. **Primary Button**: Used for finalizing sales, adding entries, etc. Full width, rounded, `48–52dp` high. Restrict to one primary button focus per screen.
4. **Floating Action Button (FAB)**: Always placed in bottom-right, distinctly identifying creation actions (e.g., adding transactions).
5. **Cards**: Most utilized surface. All Customer strips, Product instances, Alerts, and Dashboard summaries use uniformly spaced cards.
6. **Icons**: Strictly use simple, recognizable outlines (Material Icons/Iconsax). Example: ₹ for money, 📦 for inventory, 👤 for customer. No abstract replacements.
