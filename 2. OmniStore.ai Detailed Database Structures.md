# 2. OmniStore.Ai Detailed Database Structures

Version 1 – MVP Focus (Powered by Isar Database)

## 🧠 Database Architecture Philosophy

The database strictly utilizes an **Event-Based Data Model**. Instead of constantly overwriting the final values (like total stock quantity or a customer's total balance), the database records every **event** that modifies those numbers.

**3 Main Entity Groups:**
1. **Core Entities:** Stable profile data (Businesses, Users, Customers, Products, Categories).
2. **Event Entities:** Ledgers of change (Inventory Events, Transactions, Payment Events).
3. **System Entities:** Internal mechanisms (Sync logs, Device tables).

**Keys & Rules:**
- **Calculated Fields (Never Stored Directly):**
  - Initial *Stock* = SUM(`inventory_event.quantity`) for a specific `product_id`.
  - Initial *Customer Balance* = SUM(`sales transactions`) – SUM(`payment_events`).
- This guarantees conflict-resistant offline syncing and a permanent, uncorruptible audit history.

---

## 🔹 1. BUSINESS / SHOP TABLE (`business`)
*Core Entity defining the shop.*
- **`id`** (Required): Unique Business ID.
- **`business_name`** (Required): Shop Name.
- **`alias_name`**: Short Name.
- **`address`, `city`, `state`, `country`** (Required)
- **`phone`** (Required), **`email`**
- **`currency`, `language`** (Required)
- **`business_type`**: Grocery, dairy, etc.
- **`credit_enabled`**: Boolean flag.
- **`created_at`, `updated_at`**

## 🔹 2. CUSTOMER TABLE (`customer`)
*Core Entity defining the buyers.*
- **`id`** (Required)
- **`business_id`** (Required)
- **`name`** (Required)
- **`phone`, `address`**
- **`credit_limit`**
- **`trust_score`**: AI-generated score based on payment history.
- **`notes`**
- **`created_at`**
*(Note: Current balance is NOT stored here).*

## 🔹 3. PRODUCT TABLE (`product`)
*Core Entity defining items to sell.*
- **`id`** (Required)
- **`business_id`** (Required)
- **`category_id`** (Required)
- **`name`** (Required)
- **`sku`, `barcode`**
- **`unit`** (Required): kg, pcs, liter.
- **`selling_price`, `purchase_price`**
- **`is_active`** (Required)
- **`created_at`**

## 🔹 4. CATEGORY TABLE (`category`)
*Core Entity defining product groupings.*
- **`id`** (Required)
- **`business_id`** (Required)
- **`name`** (Required)
- **`color`**, **`created_at`**

## 🔹 5. INVENTORY EVENT TABLE (`inventory_event`)
*(Crucial: Every stock change creates an event here).*
- **`id`** (Required)
- **`business_id`, `product_id`** (Required)
- **`event_type`** (Required): purchase, sale, return, damage, adjustment, initial.
- **`quantity`** (Required): Can be positive (+10 purchase) or negative (-2 sale).
- **`reason`, `reference_id`** (Used for auditing).
- **`timestamp`**, **`device_id`**, **`sync_status`**

## 🔹 6. TRANSACTION TABLE (`transaction`)
*(For recording sales, returns, and total amounts).*
- **`id`** (Required)
- **`business_id`** (Required)
- **`customer_id`** (Optional: Null for walk-ins)
- **`transaction_type`** (Required): sale, payment, return, adjustment.
- **`total_amount`** (Required)
- **`discount`, `tax`**
- **`note`**
- **`timestamp`**, **`device_id`**, **`sync_status`**

## 🔹 7. TRANSACTION ITEM TABLE (`transaction_item`)
*(The line-items of products inside a `sale` transaction).*
- **`id`** (Required)
- **`transaction_id`** (Required)
- **`product_id`** (Required)
- **`quantity`** (Required)
- **`price`** (Required): Price *at the time of sale.*
- **`subtotal`** (Required)

## 🔹 8. PAYMENT METHOD TABLE (`payment_method`)
- **`id`**, **`business_id`**
- **`name`**: cash, UPI, card, udhar.
- **`is_default`**

## 🔹 9. PAYMENT EVENT TABLE (`payment_event`)
*(Whenever money actually moves, to offset customer balances).*
- **`id`**
- **`transaction_id`**
- **`method_id`**
- **`amount`**
- **`timestamp`**

## 🔹 10. SYSTEM TABLES
- **SYNC LOG TABLE (`sync_log`)**: `id`, `entity`, `entity_id`, `status`, `last_attempt`, `error`.
- **DEVICE TABLE (`device`)**: For multi-device logic tracking.
- **AI LOG TABLE (`ai_log`)**: User query strings, parsed intentions, confirmation status.
- **USER TABLE (`user`)**: Owner/manager/staff permissions (future use).
